Extended metadata quality assessment for mandatory elements

Building on our foundational minimum mandatory work, the MSU took a deeper dive into four of our mandatory elements (Type of Resource, Identifier, Genre, and Date) to assess quality criteria above and beyond the minimum required for digitized items.

  • Green is a passing score
  • Yellow is a partial score (i.e. some but not all of the quality criteria were met)
  • Orange is a failing score (i.e. the element is not present at the item-level)

What do we mean by metadata quality?

Our extended mandatory element assessment encompasses the following metadata quality criteria:

  • Completeness: Is the element or attribute present?
  • Accessibility: Can it be understood by a human and/or a machine?
  • Conformance: Does it meet core needs of our user community by conforming to expected standards?

Why is this important?

This information provides a more granular analysis to help the MSU strategize and track targeted remediation projects. Improving metadata quality improves discovery of and context for our online collections.

Extended mandatory element scoring criteria

typeOfResource

  • Exactly one typeOfResource element is marked primary (Completeness)
  • Each typeOfResource is distinct (no duplicates) (Conformance)

identifier

  • Each record with a TMS ID must have a TMS Object Number (and vice versa) (Completeness)
  • Each record with an MSS ID should have an Archives EAD ID (if described in a finding aid) (Completeness)

genre

  • Each genre element has an authority attribute that is not blank (Accessibility)
  • Each genre element has valueURI attribute that is not blank)

date

  • Each date element is w3cdtf encoded (unless BCE date) (Accessibility)
  • Record has exactly one date marked "keyDate" (Completeness)
